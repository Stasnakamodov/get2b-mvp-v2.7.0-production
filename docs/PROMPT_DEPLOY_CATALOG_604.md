**Проблема**: в базе сейчас 1485 старых товаров. Sync делает upsert по `id` — новые товары добавятся, но **старые не удалятся** (у них другие UUID). Нужно решить:

**Вариант А** — Полная очистка + заливка:
```sql

TRUNCATE catalog_products CASCADE;
TRUNCATE catalog_suppliers CASCADE;
TRUNCATE catalog_categories CASCADE;
-- Потом sync-catalog.mjs зальёт чистые 604
```

**Вариант Б** — Деактивировать старые, залить новые:
```sql
UPDATE catalog_products SET is_active = false;
-- Потом sync-catalog.mjs зальёт 604 активных
```

Рекомендация: **Вариант А** — чистый старт. Старые данные всё равно мусор.
